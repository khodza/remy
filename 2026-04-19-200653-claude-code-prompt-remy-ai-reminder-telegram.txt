     Part A — Backend HTTP API (required before frontend)

     A.1 Auth model: JWT exchange (your choice)

     Flow:
     1. Frontend sends POST /auth/telegram with Authorization: tma <initDataRaw>
      — one call at app launch.
     2. Backend validates initData signature + freshness, upserts user, returns
     { token, user, expiresAt }.
     3. Frontend stores token in memory (not localStorage — the Mini App session
      dies with the webview anyway) and sends Authorization: Bearer <jwt> on
     every subsequent request.
     4. JWT lifetime 15 minutes, rotated by calling /auth/telegram again with a
     fresh initData on 401. Keep JWT short so a compromised token has tiny blast
      radius.
     5. JWT payload: { sub: <internal user id>, tgId: <telegram user id>, iat, 
     exp }.

     A.2 Packages to add (backend package.json)

     @nestjs/jwt             (JWT signing/verification)
     @nestjs/passport + passport + passport-jwt  (guards)
     @telegram-apps/init-data-node               (initData HMAC + Ed25519
     verification — supports Bot API 8.0 public-key path)
     class-validator class-transformer           (DTO validation)

  Auth flow

  Authorization: tma <initDataRaw> → validate() HMAC via @tma.js/init-data-node
  (freshness from INIT_DATA_MAX_AGE_SECONDS, default 24h) → upsert via
  EnsureUserUsecase → JWT {sub, tgId} signed with JWT_SECRET. Subsequent calls
  use Authorization: Bearer <jwt>.

  Error mapping (global HttpExceptionFilter)

  InvalidInputError → 400 · {Task,User}NotFoundError → 404 ·
  {Parsing,Transcription}FailedError → 502 · missing/tampered/expired auth → 401
   · ownership mismatch → 403 · unknown → 500.
